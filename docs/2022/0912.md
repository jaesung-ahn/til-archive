# 0912 ~ 0918

## 0913 - # HTTP의 진화 과정
- HTTP/1.1: ASCII over TCP
- HTTP/2: Binary Multiplexed over SPDY(TCP)
- HTTP/3: Binary over Multiplexed QUIC(UDP)

### HTTP/1.1
- 연결 상태 유지
- 파이프라이닝
- HOLB 문제
- 헤더 중복 문제
- HTTPS/RESTful API의 탄생

### HTTP/2
- TCP 기반의 SPDY 프로토콜 -> HTTPS 강제
- 이진 프로토콜(바이너리 프레이밍)
- 스트림, 메시지, 프레임으로 구분
- 멀티플렉싱(응답 다중화) -> HOLB 문제 해결
- HPACK -> 헤더 중복 문제 해결
- 서버 푸시
- 스트림 별 우선 순위 설정
- TCP 자체의 HOLB 문제

### HTTP/3
- UDP 기반의 QUIC 프로토콜
- 독립 스트림 -> TCP HOLB 문제 해결
- QPACK -> 헤더 압축 문제 개선
- 캐싱을 이용한 ORTT
- 서버 푸시
- 스트림 별 우선 순위
- Connection ID 기반
- TLS -> HTTPS 강제
